GPS Cycling Tracking Software Options Review

Before I got my Garmin Edge 705 I was using the iPhone + RunKeeper combo. RunKeeper was for simply tracking the distances I was riding before I had a bike computer rather than for any kind of analysis. I get much more data out of the Garmin and it’s lead me head first into a category of software I hadn’t previously explored.

The options are numerous both online and client side but each have their own sets of positives, negatives and quirks. I’ll attempt to summarise my findings over the last week.

RunKeeper

The website presents the relatively small amount of data RunKeeper collects well. Every other app here has features that stretch beyond what RunKeeper’s can provide. If all you have is an iPhone this is without a doubt the way to go.

Positives

  • Great graph and map integration. Hover over the graph and it shows you the point on the map.
  • Great looking, readable graphs.
  • Good presentation of summary data like total distance, average speed, etc.
  • No need to upload anything, taken care of by the iPhone app.
  • Nice simple date based method of managing rides.
  • Being web based, easy to share.

Negatives

  • Inflexible. Small map that you can’t make bigger without hiding the graph.
  • Bad map presentation, in Map view it’s difficult to make out the path.
  • You have to use an iPhone. This is the biggest negative. Starting, stopping and pausing is cumbersome as the phone is in a pocket and often in a protective bag. The GPS also churns through the battery, it’ll completely drain on a 3 hour ride.

Garmin Connect

I was wary of Garmin Connect. Their client side software sucks and I thought Connect would be not much more than a bullet point on Garmin product’s feature lists. My concern was unfounded, Connect is highly polished and functional. I’d go as far to say it’s one of the better web applications out there.

Positives

  • Good looking website that’s all round, easy to use and navigate.
  • Attractive, readable graphs.
  • Easy to get data from your device to Connect with their totally unobtrusive browser plugin.
  • Simple and useful reporting UI.
  • Being web based, easy to share.

Negatives

  • No map to graph interaction outside of a cumbersome Flash “player”.
  • No ability to layer graphs or compare rides.
  • Somewhat inflexible, you can’t enlarge the map or graphs.

Rubitrack

Rubitrack is well put together and organised, a major advantage. UI conventions are followed and everything works the way you expect it to. The source list works like a contextual bunch of smart folders and it’s my favourite way any of these applications organise rides. But while it does some things brilliantly, it does others poorly.

Positives

  • Well organised, Mac like UI.
  • Crystal clear map that visually represents zones.

Negatives

  • Crappy graphs in many ways. No interaction with the map, visually confusing and doesn’t display precise readings on hover. In practice, they’re static jpegs.
  • Forces you to save and maintain a library file which it doesn’t automatically load on next open. I hate this convention. When I’m using an application to interact with a “library” of something, I don’t want to manage afile, I don’t want to save or open, I want the data side of things to be invisible. I can’t think of any situation where I’d want to manage multiple Rubitrack libraries simultaneously.
  • The calendar view has potential but is poorly executed.

TrailRunner

For donationware TrailRunner is surprisingly good but tries to be too much at the expense of absolutely nailing what it already does well. In some areas it’s polished to perfection, in others it almost seems broken.

Positives

  • The best looking graphs of any of the applications listed here. Far and away the most well thought out and executed. Very easy to read layered graphs.
  • Ability to select a section of the graph which it will then display averages for and display that selection on the map. A brilliant idea that no other app has implemented let alone done so well.
  • Great looking maps and seamless graph + map integration.

Negatives

  • Disorganised UI. Sometimes I will get lost and don’t know how to get back to my list of rides. Labels are messy and unclear.
  • Some functions popup confusing windows which aren’t obvious how they integrate into the rest of the application.
  • Ugly, annoying copyright banner on the bottom of the map. I assume this is a requirement for map use without a commercial license but it’s distracting.
  • The same frustrating save file model as Rubitrack.

Ascent

Ascent is the Final Cut Pro of this category of applications. Raw UI with every kind of function at your disposal. The UI is just plain ugly and acts strangely, drop down buttons act like toggle switches and new windows are opened all over the place. It’s a powerful application but I can’t look past the unpolished UI.

Positives

  • The option to dim the map background results in the best looking maps out of any application here.
  • There’s a great map view that shows your zones along the route. I don’t understand why this isn’t the default.

Negatives

  • Map and graph interaction is only possible when you have the graph popup open even though a graph exists as part of the main window.
  • Again, Ascent forces you to manage library files! Why do they do this?! Could you imagine having to manually save your iTunes library every time you added a song to a playlist or change the smallest piece of meta data.
  • I simply can’t tolerate the interface and interaction quirks. There’s one text box which opens up a modal text box when it gains focus. Frustrating to use.

Garmin Training Center

Granted it’s a “beta” but Garmin Training Center is too bad to dwell on. Don’t waste your time.

Conclusion

There is definitely no clear winner. All the client side options have the requirement for saving library files which pisses me off so much it’s a deal breaker across the board. Although each of them have their own individual pitfalls outside of the file issue. For now I’m sticking with Garmin Connect, although it lacks some features, it makes up for it by doing what it does do well. The UI is polished and and great reporting tools which I appreciate.

I’ve looked into the files that my Garmin Edge collects and they’re nothing more than simple XML files which opens it up for me to play with this data myself. I had a great time creating the graph on ride.jimwhimpey.com with Raphael, so I’m looking forward to diving into my cycling data with Graphael and perhaps complimenting Garmin Connect with my own software.
